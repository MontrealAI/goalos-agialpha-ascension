# License Decision

## Current recommendation

Do not select a final open-source license until founder and counsel approve the repository posture.

## Why

This repository contains a strategic implementation candidate, token-adjacent coordination logic, audit-candidate contracts, public claims boundaries, and commercialization-sensitive materials.

## Options

| Option | When appropriate |
|---|---|
| No license / all rights reserved | Best for initial private repository and audit handoff. |
| MIT | Best if the code is intentionally open and permissive. |
| Apache-2.0 | Best if patent grant language is desired. |
| BUSL / source-available | Best if public review is desired but production commercial use should be controlled. |
| Dual license | Best if open community and commercial licensing are both desired. |

## Current repository notice

Until a license is explicitly approved and added:

```text
All rights reserved. No license is granted except by explicit written permission.
```
