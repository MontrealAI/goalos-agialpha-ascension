## RSI mechanism

RSI is implemented as proof-backed upgrade rights. A candidate artifact cannot influence future work merely because it exists. It must survive evidence, evaluation, reviewer validation, red-team pressure, scope control, canary planning, monitoring, rollback readiness, and Selection Gate approval.

```text
observe weakness
-> generate hypotheses
-> execute parallel proof missions
-> test candidate improvements
-> review and red-team evidence
-> select only safe improvements
-> grant scoped upgrade rights
-> route verified value into future capability
-> chronicle what worked
-> improve the next mission portfolio
```
