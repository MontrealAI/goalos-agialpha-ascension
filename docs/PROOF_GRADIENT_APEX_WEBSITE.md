# The Proof Gradient — Website Apex Edition

## Public proposition

> Same engine. Same objective. Same compute. One difference: proof governance.

The Proof Gradient Challenge is a claim-bounded public test of hardened autoresearch versus GoalOS-governed autoresearch. It asks which machine discoveries are merely fast, and which survive the institutional chain required to become admissible capability:

```text
objective → experiments → failed branches → reward-hack rejection
→ independent replay → validator decision → accepted capability
→ conditional settlement → reuse in a harder mission
```

## Preservation architecture

This feature is added only to the temporary GitHub Pages artifact. The canonical source under `website/v86_actual_site/` remains untouched.

The autonomous workflow:

```text
builds canonical v86
→ adds the Proof Gradient Apex page and one homepage feature
→ runs the original v86 verifier
→ runs Proof Gradient preservation/data checks
→ runs desktop and mobile browser QA
→ rejects archives, private operator material, and unbounded claims
→ deploys GitHub Pages only when every check passes
```

## Evidence and claim boundary

The Mainnet directory is generated from the included v4.4.0 release-evidence snapshot: 48 GoalOS-created contracts, one canonical external AGIALPHA dependency, 48 recorded GoalOS verification successes, zero verification failures, and a passing 48-contract postcheck.

The equal-budget autoresearch result is synthetic. It is not empirical evidence that GoalOS, Recursive, or another system achieved the modeled result. Only a completed real public mission may support comparative performance, capability acceptance, AGIALPHA settlement, or on-chain promotion.
