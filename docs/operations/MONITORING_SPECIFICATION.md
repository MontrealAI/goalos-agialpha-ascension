# Monitoring Specification

Vendor-neutral alerts must watch ownership initiation/cancellation/acceptance, role changes, owner overrides, local/global pause, lifecycle transitions, treasury rotation, abnormal transfers, liability/reservation/free-balance changes, solvency deviations, limit breaches, duplicate settlement attempts, failed transfers, migration start/rollback/completion, shutdown request/cancel/finalization, unexpected pending ownership, code/configuration mismatch, RPC divergence, and invariant/accounting sentinel failure.

The canonical event catalog is `qa/monitoring-event-catalog.json`.
