# Incident Response

1. Detect and classify the event using `qa/monitoring-event-catalog.json`.
2. Preserve logs, transaction hashes, readbacks, and accounting snapshots.
3. Pause affected components or the global lifecycle when new risk must stop.
4. Use typed owner recovery/override paths only; do not use arbitrary executors.
5. Verify accounting after each action and keep protected liabilities segregated from free funds.
6. Enter wind-down, migration, or shutdown only through evidence-bearing lifecycle commands.
7. Regenerate `qa/mainnet-readiness/production-readiness.json`; do not manually overclaim public status.
