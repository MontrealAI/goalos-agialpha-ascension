# Public Governance Deployment Approval

A deployment package is publicly authorized when the PR is merged by an authorized repository owner/admin, required checks pass, branch protection or public risk acceptance exists, CODEOWNERS or sensitive-path review policy exists, README/status docs are updated, and a public release/tag `mainnet-authorization-vX.Y.Z` is created.

Release notes must state: not externally audited; not deployed; authorized for manual gated Ethereum Mainnet deployment; deployment requires runtime RPC/key; no new AGIALPHA token is deployed; AGIALPHA token address is fixed.
