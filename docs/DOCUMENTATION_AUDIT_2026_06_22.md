# Documentation Audit — 2026-06-22

## Scope

Reviewed public Markdown, generated registry outputs, release documentation, package scripts, app/website status data, badges, and operator/developer runbooks relevant to Ethereum Mainnet public status.

## Classification model

`CURRENT`, `GENERATED`, `HISTORICAL`, `LEGACY_COMPATIBILITY`, `SUPERSEDED`, `OPERATOR_ONLY`.

## Dispositions

- README: CURRENT, rebuilt as executive landing page.
- `docs/ETHEREUM_MAINNET.md`: CURRENT, canonical hub.
- Generated registry outputs: GENERATED.
- Stage-A/predeployment materials: HISTORICAL or LEGACY_COMPATIBILITY; preserved as provenance, not current guidance.
- Local-gated deployment commands: OPERATOR_ONLY/LEGACY_COMPATIBILITY; not presented as normal entry points.
- Stale claims such as predeployment Mainnet-deployed-NO wording, obsolete Solidity-version wording, and Stage-A certificate sole-source wording were removed from current docs or retained only in historical/evidence paths.

## Historical contradictions intentionally preserved

Immutable evidence and archived pre-mainnet documents may still describe predeployment states. They are preserved for provenance and must carry or be covered by historical classification.
