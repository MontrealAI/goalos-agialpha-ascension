# GitHub Repository Settings

Use these values when creating the repository through the GitHub web UI.

## Repository name

```text
goalos-agialpha-ascension
```

## Owner

Recommended:

```text
MontrealAI
```

## Description

```text
GoalOS-native reimplementation of α‑AGI Ascension using AGIALPHA for proof-settled AI workflow work.
```

## Visibility

Recommended first state:

```text
Private
```

Use public only after:

```text
public claims review
secret scan
paid product file check
legal/token counsel review for public wording
main README status verified
```

## Initialize options

Recommended for web upload workflow:

```text
Add README: no
Add .gitignore: no
Choose license: no
```

Reason: upload the prepared repository pack as the first commit.

## Website field

Leave blank at creation, or later use the official GitHub Pages / documentation URL after public-claims review.

## First two commit messages

```text
Initialize GoalOS AGIALPHA Ascension repository
```

```text
Add Ethereum audit-candidate implementation and evidence-ready docs
```

## Production-continuation hardening

After upload, enable:

```text
secret scanning / push protection where available
branch ruleset for main
pull-request review before merge
CODEOWNER review for sensitive paths
status checks before merge
restricted Actions permissions
```

Keep the repository private until public-claims review and secret/product-file audit pass.
